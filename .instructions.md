# language-adapter-sdk Instructions

Read the `typescript-library-engineering` and `canonical-ir-and-stable-identity` skills before making changes here.

- Tests go in `src/__tests__/` or co-located `*.test.ts` files.
- Coverage must stay ≥ 80% (statements, branches, functions, lines).
- Run `npx vitest run --coverage` from `packages/language-adapter-sdk` to verify.
- This package defines **contracts and types** consumed by all adapters — avoid breaking changes to exported interfaces.
- Canonical IR types (`CanonicalNode`, `CanonicalEdge`, etc.) must remain language-agnostic.
- Adapter contract changes require corresponding conformance test updates.

## File patterns

- **`*.types.ts`** — Pure type-only files (interfaces, type aliases, discriminated unions). Excluded from coverage. Put new types here, not in runtime files.
- **Barrel `index.ts`** — Thin re-exports only. Import from `*.types.ts` and runtime modules.
- Key type files: `contract.types.ts` (adapter contract, diagnostics, edits), `structural.types.ts` (nodes, edges, `SourceRange`), `behavioural.types.ts` (CFG/data-flow), `trace-links.types.ts`.

## Error tolerance types

- `ExtractionDiagnostic` with typed `ExtractionDiagnosticCode` — never plain strings.
- `NodeCompleteness` (`"complete" | "partial" | "stub"`) on `CanonicalNodeBase`.
- With `exactOptionalPropertyTypes`, omit optional properties instead of assigning `undefined`.
